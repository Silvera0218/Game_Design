## 团队构成
- **叙事导演** — 故事情节、角色设计、对话策略、叙事愿景
- **作家** — 对话写作、故事条目、物品描述、游戏内文本
- **世界建造者** — 世界规则、派系设计、历史、地理、环境故事讲述
- **艺术总监** — 角色视觉设计、环境视觉叙事、cutscene/cinematic 色调
- **关卡设计器** — 服务于叙事、节奏、环境叙事节拍的关卡布局

## 如何委派

使用 Task 工具将每个团队成员生成为子代理：
- `subagent_type: narrative-director` — 故事情节、角色设计、叙事视觉
- `subagent_type: writer` — 对话写作、传说条目、游戏内文本
- `subagent_type: world-builder` — 世界规则、派系设计、历史、地理
- `subagent_type: art-director` — 角色视觉轮廓、环境视觉叙事、电影基调
- `subagent_type: level-designer` — 服务于叙事、节奏的关卡布局
- `subagent_type: localization-lead` — i18n 验证、字符串密钥合规性、翻译余量

始终在每个特工的提示中提供完整的上下文（叙述简介、背景依赖关系、角色简介）。在管道允许的情况下并行启动独立代理（例如，第 2 阶段代理可以同时运行）。

## Pipeline

### 第一阶段：叙事方向
委托给**叙事导演**：
- 定义该内容的叙事目的：它提供什么故事节奏？
- 确定所涉及的角色、他们的动机以及这如何适应整体弧线
- 设定情绪基调和节奏目标
- 指定任何传说依赖性或引入的新传说
- 输出：带有故事要求的叙述概要

### 第二阶段：世界基金会（平行）
并行委托 — 在等待任何结果之前同时发出所有三个 Task 调用：
- **世界构建器**：创建或更新与此内容相关的派系、位置和历史的传说条目。与现有的矛盾知识进行交叉引用。为新条目设置规范级别。
- **作家**：使用语音配置文件起草角色对话。确保所有行都在 120 个字符以下，对变量使用命名占位符，并且已做好本地化准备。
- **艺术总监**：定义此内容中出现的关键角色的角色视觉设计方向（轮廓、视觉原型、显着特征）。为每个关键空间指定环境视觉叙事元素（道具构成、灯光说明、空间布置）。为任何过场动画或脚本序列定义色调调色板和电影方向。

### 第三阶段：关卡叙事整合
委托给**关卡设计师**：
- 回顾叙述概要和传说基础
- 设计关卡中的环境叙事元素
- 放置叙事触发器、对话区和发现点
- 确保节奏服务于游戏玩法和故事

### 第四阶段：审查和一致性
委托给**叙事导演**：
- 根据角色声音配置文件查看所有对话
- 验证新条目和现有条目之间的传说一致性
- 确认叙事节奏与关卡设计一致
- 检查所有谜团是否都记录了“真实答案”

### 第 5 阶段：波兰语（并行）
并行委托：
- **作者**：最终自我审查 - 验证没有行超出对话框限制，所有文本都使用字符串键（不是原始字符串），占位符变量名称一致
- **localization-lead**：验证 i18n 合规性 - 检查字符串键命名约定，标记任何无法翻译的硬编码格式字符串，验证扩展语言的字符限制空间（German/Finnish 通常 +30%），确认文本中没有需要特定于区域设置变体的文化假设
- **世界建造者**：最终确定所有新传说条目的设定级别

## 错误恢复协议

如果任何生成的代理（通过 Task）返回 BLOCKED、错误或无法完成：

1. **立即浮出水面**：在继续依赖阶段之前向用户报告“[AgentName]：BLOCKED — [reason]”
2. **评估依赖关系**：检查后续阶段是否需要被阻止的代理的输出。如果是，则在没有用户输入的情况下不要继续越过该依赖点。
3. **通过 AskUserQuestion 提供选项**，可选择：
   - 跳过此代理并注意最终报告中的差距
   - 以更窄的范围重试
   - 停在这里并首先解决阻止程序
4. **始终生成部分报告** — 输出已完成的内容。切勿因为一名代理阻塞而放弃工作。

常见的拦截器：
- 输入文件丢失（未找到故事，GDD 不存在）→ 重定向到创建它的技能
- ADR 状态为建议 → 不实施；首先运行 `/architecture-decision`
- 范围太大 → 通过 `/create-stories` 分成两个故事
- ADR 和故事之间的指令冲突 → 冲突浮出水面，不要猜测

## 文件 Write 协议

所有文件写入（叙述文档、对话文件、传说条目）都委托给
通过 Task 生成的子代理。每个子代理强制执行“我可以写入 [path] 吗？”
协议。该编排器不直接写入文件。

## Output

一份总结报告，内容包括：叙事简要状态、故事条目 created/updated、编写的对话台词、关卡叙事整合点、一致性审查结果以及任何未解决的矛盾。

结论：**完整**——已交付叙事内容。

如果管道因依赖关系未解决而停止（例如，传说矛盾或缺少用户未解决的先决条件）：

结论：**被阻止** — [reason]
