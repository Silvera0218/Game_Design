## 翻译流程

Ren'Py 翻译通常保存在 `game/tl/<language>/`。使用启动器生成翻译文件，并保持稳定的翻译标识符。

流程：

1. 冻结或稳定原文。
2. 生成目标语言翻译。
3. 翻译对白、菜单字符串和界面字符串。
4. 配置语言名称、字体和 GUI 差异。
5. 在目标语言完整游玩关键路径。
6. 原文变化后更新并检查过时翻译。

不要通过复制整套剧情 label 来制作语言版本。

## 文本和变量

```renpy
yuri _("欢迎回来，[player_name]。")
```

翻译时保留插值变量和文本标签的语义。避免：

- 把一句话拆成多个片段再拼接。
- 根据英语词序插入名词。
- 把复数、性别和语序写死在 Python 中。
- 让翻译者猜测角色、场景或占位符。

为翻译团队提供上下文、角色口吻、变量含义和长度限制。

## 字体与版式

- 确认字体许可允许游戏分发和所需字符集。
- 为中日韩、拉丁、Cyrillic 等字符提供覆盖。
- 测试 fallback、粗体、斜体、Ruby、标点和换行。
- 为翻译文本预留扩展空间，不依赖固定宽度。
- 检查按钮、存档标题、历史记录和选择菜单的长文本。
- RTL 或复杂文字需要目标版本和平台专项验证。

## 语音和自动语音

- voice 文件与对白建立稳定映射。
- 历史记录可提供语音重播。
- self-voicing 依赖可读的文本结构和替代文本。
- 图片按钮、纯图标和关键图像应提供可朗读信息或替代操作。
- 不要把重要文字烘焙进背景图片。

## 输入与导航

- 保留鼠标、键盘和控制器的标准确认、返回、历史、跳过等操作。
- 自定义 keymap 时避免破坏系统级和可访问性快捷键。
- 所有菜单应有清晰 focus、hover、selected 和 insensitive 状态。
- 触摸目标足够大，并避开安全区。
- 拖放、滑动和手势提供替代按钮。
- 动画闪烁、快速缩放或强烈晃动应可降低或关闭。

## 验收检查

- 切换语言后主菜单和游戏菜单不混用语言。
- 变量、标签和换行显示正确。
- 字体无缺字和 tofu 方框。
- 长文本不溢出。
- 历史、存档、设置、确认框均可导航。
- self-voicing 能读出关键控件和对白。
- 自动播放、跳过、文本速度和音量可调。
- 纯音频信息有视觉替代，纯视觉关键信息有文字或语音替代。
